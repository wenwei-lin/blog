三种面向可访问性与公平的设计方法

## 1. Universal Design（通用设计）

| 关键词                | 概念                                                             |
| --------------------- | ---------------------------------------------------------------- |
| **One-size-fits-all** | 为尽可能多的人提供**单一**解决方案                               |
| **优势**              | 易于维护与推广，初期能快速覆盖大多数场景                         |
| **局限**              | “大号帽子”式思维：看似适配所有人，实则仍有边界——特殊需求常被稀释 |

**示例**

- 自动门：轮椅使用者、抱孩子进门的人都能用，但门宽、感应高度如果配置固定，仍可能难以满足极端场景。

## 2. Inclusive Design（包容性设计）

> **设计给“一人”，造福“多人”**

### 核心原则

- **Solve for one, extend to many**：先聚焦一个被忽视的用户，再将方案外溢给大众
- 维度多元：**能力 · 语言 · 年龄 · 性别 · 经济条件 · 文化**
- **“全流程包容”**：调研-原型-测试-上线都要有目标群体的声音

### 思考提问

1. 这项功能**默认前提**是什么？是否把某些群体排除在外？
2. 如果只为那个“极端用户”服务，我该怎样设计？
3. 当方案扩散到主流用户，体验会更好还是更糟？

## 3. Equity-Focused Design（聚焦公平设计）

| 概念         | Equality vs. Equity                                      |
| ------------ | -------------------------------------------------------- |
| **Equality** | 每个人同样的资源                                         |
| **Equity**   | 根据差异提供**不同强度**的资源，让所有人都能到达同一高度 |

### 实践框架

1. **定义目标**：产品想解决什么真实痛点？
2. **锁定被忽视人群**：历史上从未被好好服务的那部分用户是谁？
3. **“转正”他们的需求**：把核心流程围绕他们重构，而不只是附加选项。

**案例灵感**

- 金融科技中的“信用盲区”用户：在设计借贷产品时，为缺乏传统征信记录的人建立替代信用评估模型，而不是简单拒绝。

## 4. 三者关系 & 设计决策图

- **通用** → 起点。快速满足主流需求。
- **包容** → 升级。解决具体边缘痛点，顺带优化大众体验。
- **公平** → 深耕。主动修正结构性不公，提供差异化支持。

> **结论**：没有一种方法能“包打天下”。真正的用户体验是**动态切换**——理解背景，选对工具，再组合落地。

## 5. 写给未来的提醒

1. **需求评审时**：先问“谁可能被排除？为什么？”
2. **原型测试时**：至少找一位*目标少数*用户参与；不要只看平均数。
3. **Roadmap 排期**：把可访问性与包容性列为**非功能需求**（NFR），写进验收标准，而非“nice-to-have”。
4. **团队文化**：在设计回顾会上分享包容性案例，让“用户多样性”成为习惯，而不是节点任务。

让每次迭代都多一点包容，少一点假设——这是设计师真正的“功德值”。
